📡 I/O Communication

How computers communicate with external devices

What is I/O Communication?

Input/Output (I/O) communication is the process of transferring data between a computer and external devices. It allows computers to interact with the real world!

Why is I/O Important?

  • Without I/O, computers would be completely isolated
  • Enables user interaction and data exchange
  • Critical for all computing systems
  • Affects overall system performance

Basic Concepts

🔌 Input Devices

Keyboard, Mouse, Camera, Microphone - devices that send data to the computer

💻 Output Devices

Monitor, Printer, Speaker - devices that display or present data from computer

💾 Storage Devices

Hard Drive, SSD, USB - devices that store data for long-term use

Types of I/O Devices

Device Type Examples Purpose
Input Devices Keyboard, Mouse, Microphone, Camera Send data to computer
Output Devices Monitor, Printer, Speaker, Headphones Display/Present data
Storage Devices Hard Drive, SSD, USB, Memory Card Store and retrieve data

I/O Communication Methods

⚡ Programmed I/O

How: CPU directly checks device status

Pros: Simple to implement

Cons: Wastes CPU time

🔔 Interrupt-Driven I/O

How: Device interrupts CPU when ready

Pros: Better CPU utilization

Cons: More complex

🚀 DMA (Direct Memory Access)

How: Device transfers data directly to memory

Pros: Very fast, CPU free

Cons: Hardware complex

🌍 Unicode Encoding

Universal character representation for all languages

What is Unicode?

Unicode is an international standard that assigns a unique number (code point) to every character in every language. It supports over 149,000 characters from 200+ languages!

Key Facts:

  • Industry standard for text encoding
  • Supports all major writing systems
  • Mandatory for modern web (HTML5)
  • Range: U+0000 to U+10FFFF

History of Unicode

1989 - Unicode Created
1991 - Unicode 1.0 Released
1996-2000 - Rapid Expansion
2010-2026 - Web Standard

Unicode Encoding Formats

UTF-8

Bytes: 1-4 per character

Used In: Web, HTML, JSON

Example: A = 1 byte, 中 = 3 bytes

UTF-16

Bytes: 2-4 per character

Used In: Java, C#, Windows

Example: Good for mixed languages

UTF-32

Bytes: 4 per character

Used In: Internal processing

Example: Simple but memory-heavy

🌐 Network Architecture

Client-Server vs Peer-to-Peer Systems

Network Architecture Overview

Network architecture defines how computers connect and communicate to share resources. Two primary models dominate: Client-Server and Peer-to-Peer.

🖥️ Client-Server Architecture

One central server provides resources to multiple clients.

Characteristics:

  • ✓ Centralized control and management
  • ✓ Strong security mechanisms
  • ✓ Easy data backup and recovery
  • ✗ Single point of failure
  • ✗ Server becomes bottleneck

Real-World Examples:

✉️ Email (Gmail, Outlook)
🌐 Websites (Google, Facebook)
📁 Cloud Storage (Dropbox)
🏦 Banking Systems

💻 Peer-to-Peer (P2P) Architecture

All devices have equal capabilities and connect directly without central authority.

Characteristics:

  • ✓ No single point of failure
  • ✓ Highly scalable
  • ✓ Lower infrastructure cost
  • ✗ Complex to manage
  • ✗ Security challenges

Real-World Examples:

⚡ BitTorrent
💰 Bitcoin/Blockchain
📱 Skype (P2P Calls)
🌍 IPFS

Detailed Comparison

Feature Client-Server Peer-to-Peer
Architecture Centralized Decentralized
Server Required Yes - mandatory No - not needed
Scalability Good for many clients Excellent as network grows
Reliability Single point of failure Highly fault tolerant
Security Easier to control More challenging
Cost High (expensive servers) Low (use existing machines)
